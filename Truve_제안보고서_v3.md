# Truve
## 커뮤니티 거버넌스를 도입한 투명 기부 플랫폼

> **"We are not a donation platform. We are a governance layer for charity."**

**Korea Financial Innovation Program 2026 제출용 제안 보고서 v3**
작성 기준: 문제 정의 · XRPL 활용 방식 · 하이브리드 아키텍처 · 5주 MVP 범위 · 메인넷 PoC 방향 · 사업화 가능성

---

## 1. 보고서 목적

Truve는 기부자가 단순히 돈을 내는 사람이 아니라, 기부 생태계의 의사결정에 참여하는 주체가 되는 XRPL 기반 투명 기부 플랫폼입니다.

기존 기부 플랫폼이 "투명성 제공"에 머물렀다면, Truve는 기부 행위 자체를 **참여 자격(Proof NFT)**으로 전환하고, 커뮤니티가 직접 재단과 프로젝트를 선정하는 거버넌스 구조를 도입합니다. 장기적으로는 DAO형 Treasury 운영으로 확장 가능한 구조를 지향합니다.

본 보고서는 KFIP 2026 프로그램 제출 관점에서 핵심 문제 정의, XRPL 기술 활용 방식, 하이브리드 온/오프체인 아키텍처, 5주 MVP 범위, 메인넷 PoC 시나리오, 사업화 가능성을 중심으로 정리합니다.

## 3. 문제 정의

### 기존 기부 시스템의 3가지 구조적 한계

**1. 투명성 부재**
- 기부금 사용 내역 공개 의무 없음
- 대형 횡령 사건 반복 (사랑의열매, 정의기억연대 등)
- 기부자가 자신의 기부금 추적 불가

**2. 기부자 소외**
- 기부 후 기부자의 역할은 완전히 종료
- 기부금 사용 방향에 대한 발언권 없음
- "돈 주고 끝"인 일방향 구조

**3. 재단 선택의 어려움**
- 국내 등록 비영리단체 15,000개 이상
- 각 단체의 신뢰도·활동 내역 비교 불가
- 기부 성향 기반 추천 시스템 부재

### 핵심 인사이트

> 기존 플랫폼은 투명성을 "보여주는" 것으로 문제를 해결하려 했다.
> Truve는 기부자를 **의사결정 주체**로 만들어 구조 자체를 바꾼다.

## 4. 아키텍처: 하이브리드 온/오프체인 설계

### 설계 원칙

Truve는 기부금 흐름과 신뢰 증명을 분리합니다.

- **오프체인 (원화)**: 기부자 결제, 플랫폼 정산, 재단 송금, donor tier 계산
- **온체인 (XRPL)**: 기부 기록 해시 증명, Proof NFT 발행, Treasury 관리, 거버넌스 투표 기록

이 분리는 단순한 기술 선택이 아닙니다. **기부자 진입 장벽을 낮추고, 재단이 원화로 수취할 수 있게 하며, 국내 기부금품법 규제 체계 안에서 운영 가능하게 만드는 현실적인 구조입니다.**

### 전체 자금 흐름

```
[기부자 결제] ── 원화 (카드/계좌이체) ──▶ [Truve 플랫폼 계좌]
                                                    │
                              ┌─────────────────────┼──────────────────┐
                              ▼                     ▼                  ▼
                    [재단 배치 정산]         [운영비 확보]       [Treasury 적립]
                    원화 일괄 송금            (10%)             XRPL 온체인
                    (85%, 일/주 단위)                           (5%)
                              │
                              ▼
                    [XRPL 온체인 기록]
                    ├─ 기부 이벤트 해시 증명
                    ├─ Proof of Giving NFT 발행
                    ├─ 재단 정산 완료 해시 기록
                    └─ Treasury 트랜잭션 기록
```

### 왜 즉시 온체인 정산이 아닌가

> XRPL의 거래 수수료는 건당 약 0.00001 XRP(≈0.003원)로 사실상 무료 수준입니다.
> 그럼에도 배치 정산 구조를 선택하는 이유는 **수수료가 아니라 현실적 운영 가능성** 때문입니다.

| 비교 항목    | 즉시 온체인 정산       | 배치 정산 (Truve 방식) |
| -------- | --------------- | ---------------- |
| 기부자 요구사항 | 암호화폐 지갑 필수      | 일반 카드/계좌 결제 가능   |
| 재단 수취 방식 | XRP 수취 능력 필요    | 기존 원화 계좌로 수취     |
| 규제 적용    | 암호화폐 이동 규제 가능성  | 기부금품법 기존 체계 적용   |
| 파트너십 난이도 | 블록체인 이해 재단만 가능  | 모든 기부 재단 가능      |
| 기부자 진입장벽 | 높음 (Web3 지식 필요) | 낮음 (기존 결제 방식)    |

## 6. 핵심 기능 상세

### 6-1. 배치 정산 + XRPL 증명 구조

기부금은 원화로 처리되지만, 모든 정산 이벤트는 XRPL에 해시로 기록됩니다.

| 단계 | 처리 방식 | XRPL 기록 내용 |
|------|-----------|---------------|
| 기부자 결제 완료 | 원화 수취 (오프체인) | 기부 이벤트 해시 + 타임스탬프 |
| Proof NFT 발행 | XRPL XLS-20 (온체인) | 기부자 지갑에 참여 자격 NFT 발행 |
| 재단 배치 정산 | 원화 일괄 송금 (오프체인) | 정산 완료 해시 + 금액 기록 |
| Treasury 적립 | XRPL Payment (온체인) | 적립 트랜잭션 공개 기록 |
| Treasury 집행 | XRPL Payment (온체인) | 집행 트랜잭션 공개 기록 |

### 6-2. NFT 구조: MVP에서는 역할을 2개로

Truve의 NFT는 MVP 단계에서 역할을 명확하게 2개로 제한합니다.

**온체인 (XRPL): Proof of Giving NFT 1종**

- 기부 완료 시 자동 발행 (XLS-20)
- 기부 영수증 역할 — 언제, 얼마를, 어느 재단에 기부했는지 증명
- 거버넌스 투표 참여 자격

**오프체인: donor tier (누적 기부 기반)**

| 등급 | 조건 | 투표 가중치 |
|------|------|------------|
| Supporter | 1회 이상 기부 | 1표 |
| Builder | 누적 기부 10만 원 이상 | 2표 |
| Steward | 누적 기부 50만 원 이상 | 3표 (상한) |

> 투표 가중치 상한(cap)을 3표로 제한하여 고래 독점을 방지합니다.
> 개인 최대 3표 구조는 Phase 2 Quadratic Voting으로 자연스럽게 확장됩니다.

**MVP에서 제외 (Phase 2)**
- Bronze/Silver/Gold/Platinum 온체인 레벨 민팅
- NFT 혜택 차등 시스템
- NFT 2차 거래 마켓플레이스
- 발의권 및 위원회 자격

### 6-3. 재단 태그 시스템

기부자가 자신의 성향에 맞는 재단을 쉽게 찾을 수 있도록 태그 기반 분류를 제공합니다.

**태그 예시:** `#아동` `#환경` `#동물` `#긴급구호` `#교육` `#노인` `#장애` `#해외원조`

- 태그 기반 필터링으로 재단 탐색 (5주 MVP 구현)
- "당신은 환경 + 동물 성향의 기부자입니다" AI 추천 (Phase 2)

### 6-4. Treasury 거버넌스

Treasury는 커뮤니티가 직접 운영하는 온체인 공공 자금입니다.
기부금의 5%가 자동 적립되며, 잔액과 입출금 내역이 XRPL에 완전 공개됩니다.

**운영 원칙**

> 플랫폼 수수료의 일부를 커뮤니티 Treasury로 재배분합니다.
> Treasury는 플랫폼 이익이 아니라, 추가 공익 집행을 위한 공동 기금입니다.

**거버넌스 흐름**

```
Proof NFT 보유자 → 재단/프로젝트 투표 참여
         ↓
donor tier 기반 가중치 적용 (1~3표, cap 존재)
         ↓
투표 결과 XRPL 온체인 기록
         ↓
선정된 재단/프로젝트에 Treasury 집행
         ↓
집행 내역 XRPL 온체인 영구 기록
```

**Phase 2: Quadratic Voting 도입**

```
현재 (MVP):     누적 기부액 → 1~3표 (cap)
Phase 2:        누적 기부액 → √누적기부액 (Quadratic)
```

> "We prevent whale dominance through quadratic governance."

## 7. 왜 XRPL인가

Truve에서 XRPL은 기부금의 이동 레일이 아닙니다. **기부 흐름의 진실성을 보장하고, 기부자에게 참여 권한을 돌려주는 신뢰 레이어**입니다.

| XRPL 특성 | Truve 적용 가치 |
|-----------|----------------|
| 건당 수수료 0.00001 XRP (≈0.003원) | Treasury 적립·집행, NFT 발행 비용 사실상 무료 |
| NFT (XLS-20) 표준 내장 | 별도 스마트컨트랙트 없이 Proof NFT 발행 |
| 불변 원장 기록 | 기부 이벤트 해시, Treasury 입출금 영구 공개 |
| Escrow 네이티브 지원 | 캠페인 목표 미달 시 자동 환불 (Phase 2) |
| 빠른 트랜잭션 (3~5초) | NFT 발행, Treasury 기록 즉각 처리 |

### 타 체인과의 비교

| 비교 항목 | Ethereum | Solana | **XRPL** |
|-----------|----------|--------|----------|
| 거래 수수료 | 높음 (Gas) | 낮음 | **극저 (0.00001 XRP)** |
| NFT 표준 | ERC-721 (외부 컨트랙트) | Metaplex | **XLS-20 (네이티브)** |
| Escrow | 별도 구현 필요 | 별도 구현 필요 | **네이티브 지원** |
| 운영 안정성 | 높음 | 중간 | **높음 (10년+)** |

## 8. 수익 모델

### 기부금 분배 구조

```
기부금 100%
├── 85%   →  재단 (원화 배치 정산, 일/주 단위)
├── 10%   →  플랫폼 운영비 (오프체인)
```

> 플랫폼 수수료의 일부를 커뮤니티 Treasury로 재배분합니다.
> Treasury는 플랫폼 이익이 아니라, 추가 공익 집행을 위한 공동 기금입니다.

## 10. Final Pitch Day PoC 시나리오

> Testnet으로 전체 흐름을 보여주고,
> 핵심 2개 기능(기부 해시 + Proof NFT)은 Mainnet에서 실시간 시연합니다.

**시연 순서**

1. 기부자 앱에서 `#환경` 태그 재단 선택 → 카드 결제로 기부 실행 (원화)
2. **[Mainnet]** 결제 완료 즉시 XRPL Mainnet에 기부 이벤트 해시 기록 → Explorer 실시간 확인
3. **[Mainnet]** Proof of Giving NFT 자동 발행 → 기부자 지갑 수령 확인
4. **[Testnet]** Treasury 5% 자동 적립 → 온체인 잔액 변화 확인
5. **[Testnet]** 거버넌스 투표 화면: donor tier 가중치로 다음 프로젝트 투표 시연
6. 재단 대시보드: 배치 정산 예정 내역 + 이전 정산 XRPL 기록 확인
7. 기부 이력 조회: 해시 기반 온체인 증명 전체 흐름 표시

**핵심 메시지**
> "이 플랫폼에서 기부하면, 당신의 기부는 영수증이 아니라 권한이 됩니다."

## 13. 결론

Truve는 기부 시장의 투명성 문제를 단순한 기록 공개로 해결하지 않습니다. 기부 행위 자체를 **거버넌스 참여 자격으로 전환**하는 구조를 통해, 기부자가 생태계의 의사결정 주체가 되는 투명 기부 플랫폼을 제안합니다.

기부금은 원화로 처리되어 기부자의 진입장벽을 낮추고 모든 재단이 기존 방식으로 수취할 수 있습니다. XRPL은 기부금 이동의 레일이 아니라 **신뢰 증명과 거버넌스의 레이어**로 작동합니다. 이 하이브리드 구조가 "현실적으로 런칭 가능한 서비스"라는 KFIP 심사 기준에 가장 잘 부합합니다.

5주 MVP는 핵심만 완성도 있게 구현하고, Quadratic Voting·AI 추천·온체인 레벨 NFT 등 복잡도 높은 기능은 설계를 완료한 상태로 Phase 2에 배치합니다. Final Pitch Day에서는 Mainnet에서 실제로 동작하는 기부 해시 기록과 Proof NFT 발행을 시연하여 "런칭 가능성"을 직접 증명합니다.

**Truve의 차별점 세 줄 요약**

1. 기부금은 원화로, 신뢰는 XRPL로 — 진입장벽 없는 Web3 기부
2. Proof NFT = 영수증 + 투표 자격 — Web3를 억지로 붙이지 않은 구조
3. 커뮤니티 거버넌스 — 장기적으로 DAO형 Treasury로 확장 가능한 시스템
