# Truve IA 확정안 + TS/HTML 우선순위 (v1)

작성일: 2026-04-13  
기준: 사용자 요구사항 + `truve_planner/dev/qa` 방향 + 병렬 에이전트 검토 결과

## 1) IA 확정 (프로토타입 기준)

### 상위 탭 (1차)
1. 기부 담기
2. 재단 탐색
3. Truve 설명
4. 내 기부 Status

### 커뮤니티 배치 결정
- 프로토타입(MVP): `재단 탐색 > 재단 상세 > 커뮤니티` 하위 구조로 시작
- 확장(Phase 2): 조건 충족 시 상위 탭 `커뮤니티` 독립

결정 이유:
- 초기 운영 리스크(허위정보, 분쟁, 모더레이션 폭증) 최소화
- 재단 맥락 기반 대화로 신뢰도 확보
- IA/개발 복잡도 통제

## 2) 탭별 목적/핵심 컴포넌트/핵심 액션

| 탭 | 목적 | 핵심 컴포넌트 | 핵심 사용자 액션 | MVP | Phase 2 |
|---|---|---|---|---|---|
| 기부 담기 | 책가방/ETF형 묶음 기부 | 재단 카드, 번들 카드, 비율 슬라이더, 장바구니, 최종확인 모달 | 담기, 비율 조정, 총액 입력, 기부 실행 | 기본 번들 3종 + 수동 비율 | 정기기부, 자동 리밸런싱, 개인화 추천 |
| 재단 탐색 | 재단 검색/비교/상세 | 검색, 태그 필터, 리스트/그리드, 신뢰지표 카드, 상세 모달 | 탐색, 비교, 상세 확인, 담기 진입 | 태그 필터 + 상세 + 담기 버튼 | 비교표, 즐겨찾기, 맞춤 정렬 |
| Truve 설명 | 구조 신뢰 전달 | 오프체인/온체인 흐름도, Proof NFT 설명, FAQ | 구조 이해, FAQ 확인, 기부 탭 이동 | 정적 설명 + FAQ | 인터랙티브 설명 |
| 내 기부 Status | 내 기부 추적/증빙 | 상태 타임라인, 거래내역, 누적 지표, 증빙 링크 | 상태 확인, 증빙 열람, 재기부 | 타임라인 + 증빙 링크 | 기간 리포트, 배지/티어 강화 |

## 3) 커뮤니티 구조 로드맵

### Phase 1 (프로토타입)
- 재단별 커뮤니티만 제공 (공지/업데이트/댓글 최소 기능)
- 재단 상세 내부 진입만 허용
- 신고/관리자 고정공지/외부링크 정책 적용

### Phase 2 (성장)
- 공통 라운지(읽기 중심) 추가
- 신뢰지표 중심 랭킹 노출

### Phase 3 (확장)
- 상위 탭 `커뮤니티` 독립
- 재단 채널 + 공통 채널 + 거버넌스 토론 채널 운영

### 상위 탭 분리 트리거(정량)
1. WAU(주간 활성 사용자) 안정 증가
2. 신고 처리 SLA 달성
3. 허위정보 재발률 안정
4. 특정 재단 편중 완화

## 4) TS/HTML 실행 우선순위 (1~2주)

## Week 1 (핵심 플로우)
1. T01 IA/라우팅 확정: 4개 탭 기본 HTML 스켈레톤
2. T03 최소 전역 상태: `cart`, `allocation`, `userStatus`, `filters`
3. T04 도메인 타입/모의데이터: 재단/번들/유저/기부이력
4. T05 API 추상화: `FoundationRepo`, `DonationRepo`, `UserRepo` + mock adapter
5. T06 기부 담기 MVP: 담기/비율조정/합계검증/최종확인
6. T07 재단 탐색 MVP: 검색/필터/상세/담기 진입
7. T08 핵심 시나리오 테스트 문서화

## Week 2 (신뢰/설명/상태 + 커뮤니티 최소)
1. T09 내 기부 Status 타임라인
2. T10 Truve 설명 탭 고도화
3. T11 공통 컴포넌트 확장 (BundleCard/TrustBadge 등)
4. T12 URL 쿼리 기반 필터 상태 동기화
5. T13 mock network (지연/실패/재시도)
6. T14 재단 하위 커뮤니티(읽기 전용)
7. T15 회귀 + 접근성/반응형 점검

## 5) 오늘 착수 순서

1. T01 (탭 IA + 라우팅 스켈레톤)
2. T03 (스토어 최소 구조)
3. T04 (도메인 타입/모의데이터)
4. T05 (API 추상화)

## 6) MVP 성공 기준 (데모 관점)

1. 기부 담기에서 3개 재단/번들을 비율로 담아 총합 100% 검증 가능
2. 최종확인에서 예상 분배 금액이 정확히 계산됨
3. 내 기부 Status에서 최소 1건의 단계 타임라인 표시 가능
4. 재단 탐색에서 태그 필터 + 상세 진입 + 담기 진입이 끊김 없이 동작
