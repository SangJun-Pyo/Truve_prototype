# Truve 프로토타입 기획서 v1

작성일: 2026-04-19  
문서 목적: 현재 구현된 Truve 웹 프로토타입의 목표, 구조, 기능, 기술 방향, 향후 확장 계획을 정리한다.

3. XRPL/Xaman 연동 가능성을 데모 레벨에서 검증

## 2. 핵심 가치 제안

1. 기부의 사용성: 재단을 개별 선택하거나 ETF형 번들로 한 번에 담아 비율 조정
2. 기부의 신뢰성: 기부 증빙(Proof NFT, 트랜잭션) 기반 상태 확인
3. 기부의 참여성: NFT 보유자를 중심으로 거버넌스 투표 참여

---

## 3. 정보 구조(IA)

## 4. 구현 기능 요약

### 4.1 기부 담기 탭
1. 재단 목록/ETF형 번들 목록 조회
2. 책가방 담기/삭제
3. 비율 슬라이더 조정
4. 비율 합계 100% 검증
5. 예상 분배 미리보기
6. Xaman SignIn 연결
7. Xaman Payment 요청 기반 기부 실행(테스트넷 데모 플로우)

### 4.2 재단 탐색 탭
1. 재단 검색
2. 카테고리 필터
3. 재단 카드(설명/신뢰지표) 표시
4. “기부 담기에 추가” 동선 제공

### 4.3 거버넌스 투표 탭
1. Proof NFT 보유 여부 기반 투표 가능/불가 표시
2. 사용자 티어 가중치(Seed/Sprout/Forest) 반영
3. 후보 재단 선택 투표
4. 투표 결과 실시간 반영(로컬 스토리지 기반 목업)

### 4.4 Truve 설명 탭
1. 오프체인 결제 + 온체인 증빙 구조 설명
2. Proof NFT와 거버넌스 연결 컨셉 요약

### 4.5 내 기부 Status 탭
1. 사용자 요약 정보(누적 기부, 등급)
2. 최근 기부 타임라인
3. 기부 내역/증빙 링크 표시

## 5. 데이터/도메인 설계(프로토타입)

주요 Mock 데이터:
1. `src/mocks/foundations.json`
2. `src/mocks/bundles.json`
3. `src/mocks/donations.json`
4. `src/mocks/users.json`

포함 개념:
1. 재단 정보(카테고리, 지역, 설명, 태그, 신뢰지표)
2. 번들 포트폴리오(재단 비율 구성)
3. 기부 이력(결제/증빙/NFT/정산 상태)
4. 사용자 티어 정보

특이사항:
- `Truve 커뮤니티 펀드`를 재단 목록에 포함해 플랫폼 자체 후원 트랙을 제공

## 6. 기술 구성

### 6.1 프론트엔드
1. HTML + TypeScript + CSS
2. Vite 기반 로컬 개발 서버

### 6.2 API 레이어(목업)
1. Repository 인터페이스 분리
2. Mock Adapter로 데이터 공급
3. 향후 실 API 교체를 고려한 구조

### 6.3 XRPL/Xaman
1. `xrpl.js` 사용 (`xrpToDrops` 등)
2. Xaman Platform API로 SignIn/Payment payload 생성
3. 프로토타입에서는 테스트넷 전제

---

## 7. 사용자 플로우

### 7.1 기부 플로우
1. 재단/번들 선택
2. 비율 조정(100%)
3. 금액 입력
4. Xaman 지갑 SignIn 연결
5. Payment 승인
6. 기부 완료 메시지 및 트랜잭션 정보 확인

### 7.2 거버넌스 플로우
1. NFT 보유 여부 확인
2. 티어 가중치 안내
3. 후보 재단 선택 투표
4. 결과 반영

---

## 8. 현재 제약 사항

1. 프로토타입 특성상 백엔드 영속 DB 미적용
2. 일부 상태는 로컬 스토리지/모의 데이터 기반
3. Xaman API Key/Secret을 화면 입력으로 처리(운영용 아키텍처 아님)
4. 실운영 보안/권한/감사 체계는 미구현

---

## 9. 향후 고도화 방향

### 9.1 기술
1. 프론트에서 Xaman 비밀키 제거, 백엔드 프록시 전환
2. 실시간 트랜잭션/잔액 조회 API 연동

4. 번들 추천/정기기부/리밸런싱 기능 추가

### 9.3 운영

3. 규제 준수 검토(기부금품/전자금융 관련)

## 11. 결론

본 프로토타입은 Truve의 핵심 컨셉인  
`기부 실행성 + 증빙 기반 신뢰 + 참여형 거버넌스`를  
단일 웹 UX로 시각화하고 검증하는 데 초점을 맞췄다.

다음 단계는 보안/백엔드/실운영 정책을 보강해  
데모 수준의 흐름을 서비스 수준의 안정성과 투명성으로 확장하는 것이다.
